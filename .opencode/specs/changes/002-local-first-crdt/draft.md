# Local-first подход с разрешением конфилков (CRDT)

Задача достаточно интересная - реализация local first-подхода с точки зрения резолва конфликтов и формата хранения самой игры. Здесь прям челлендж на счет того, как правильно хранить данные.

Сейчас поле игры хранится в виде массива байтов (arraybuffer). Для резолва конфликтов (CRDT) нужно хранить её в каком-нибудь формате типа yjs.

Хранить небольшие поля нет проблем, то есть мы просто создаем yjs объект с нашим стейтом приложения, и синкаем между различными инстансами. Челлендж включается в том, чтобы реализовать правильное хранение больших данных, например, 100 Мбайт array buffer в формате yjs.

Есть идея не хранить в yjs все поле, а в yjs хранить только операции клика по клетке (правой и левой кнопки мыши), а так же изначальная инициализация поля (передаем только seed и другие стартовые параметры). По сути модель данных меняется на сгенерированное поле (на клиенте) и массив действий, чтобы другой клиент мог себе собрать актуальное состояние путем мержа локальных и внешних действий.

То есть, условно, у нас есть два инстанса: в одном откликаем, он синкается с другим. На текущий момент нужно будет это всё проверить в рамках одного приложения. В будущем это будет работать через WebSocket либо WebRTC.
