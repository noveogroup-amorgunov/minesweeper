## Критические точки внимания

### 1. Порядок инициализации в GameRoom
```
constructor:
  1. Создать GamePlayer
  2. Создать CrdtManager (с onExternalOperations callback)
  3. Создать GameEngine с mode: 'seeded'
  4. Подключить SyncProvider (если roomId передан)

joinGame:
  1. Подключиться к документу (получить операции)
  2. Найти первую join операцию
  3. GameEngine.restart() с параметрами из join
  4. Добавить свою join операцию
  5. Применить все leftClick/rightClick
```

### 2. Фильтрация в CrdtManager
- Всегда использовать `transaction.origin !== 'local'` для отличия внешних операций
- Callback `onExternalOperations` вызывать только для внешних операций

### 3. Дедупликация в GameRoom
- Использовать `Set<string>` для отслеживания обработанных операций
- Ключ: `${type}-${x}-${y}` для click операций
- Проверять перед вызовом `gameEngine.applyOperation()`

### 4. Параметры игры
- Всегда брать из первой `join` операции, а не из JSON metadata
- Seed = roomId для детерминированной генерации
